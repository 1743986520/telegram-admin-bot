# Telegram Admin Bot 🤖

一個用於 Telegram 群組管理的機器人，支持廣告自動攔截、圖片驗證碼、炸群應急鎖群、防護模式、成員管理、投票提案等功能，基於 `python-telegram-bot` 開發。

## 功能特點

- 🚫 **本地廣告攔截**：正則規則 + TF-IDF 語義相似度 + 廣告模板庫，自動禁言並通知管理員
- 🧠 **動態廣告樣本庫**：可用 `/addsample` 將新廣告加入樣本庫，會自動與原生模板庫正規化去重，立即重建偵測器
- 🛡️ **誤封白樣本庫**：可用 `/whitelist` 或點擊攔截通知上的按鈕，把誤判訊息加入非廣告樣本並**自動恢復該用戶的發言權限**
- 📦 **樣本庫提取**：可用 `/exportsamples` 匯出廣告樣本與白樣本 JSON；`/cleanupads` 整理並去除與原生模板重複的動態樣本
- 🔍 **帳號畫像偵測**：入群時直接檢查用戶名／暱稱／簡介是否命中廣告模板庫，命中視為高置信度廣告帳號，**直接禁言、不提供自助驗證**，需管理員手動處理
- 🌐 **非中文名稱偵測**：暱稱完全不含中文字（純英文或其他語言）視為軟性可疑訊號，需通過圖片驗證碼
- 🧮 **圖片算術驗證碼**：軟性可疑用戶（`@標籤`、網址、非中文名稱）需點選圖片中算式的正確答案才能解除禁言，最多 3 次機會，答對/答錯/過期都會在短暫顯示結果後自動刪除題目訊息
- 🚨 **炸群應急鎖群（`/omg`）**：開啟後所有新加入成員直接禁言、不歡迎、不驗證、保持絕對安靜，並自動刪除入群系統訊息，直到 `/stop` 解除
- 🛡 **防護模式（`/guard`）**：開啟後新成員完全靜默禁言（連提示訊息都不發）並記錄名單，`/stop` 解除時可選擇「全部踢出／挑選踢出／不踢出」
- 🔇 **禁言指令**：`/ban` 管理員專用，回覆訊息或指定 ID 即可禁言
- 👋 **成員離開通知**：成員退群自動發送告別訊息
- 🗳️ **投票提案系統**：`/propose` 發起提案，群組民主決策
- 📋 **群組查詢**：`/list` 查詢 Bot 管理的所有群組
- 🛡️ **完善異常處理**：超時控制、錯誤日誌、零誤封設計

## 廣告攔截架構

```
訊息 / 入群時的用戶名・暱稱・簡介
 │
 ├─ 前處理：clean_text
 │   └─ 去除零寬字元、NFKC 全半形正規化、還原拆字混淆（+·•/\-.符號、同音／形近字表）
 │
 ├─ L1 正則規則引擎（<0.1ms，130+ 條）
 │   └─ 命中 → 禁言 + 通知管理員
 │
 ├─ L2 TF-IDF 語義相似度（~3ms）
 │   └─ 相似度超過自適應門檻 → 禁言 + 通知管理員
 │
 └─ 入群帳號畫像（用戶名 / 暱稱 / 簡介）
     ├─ 命中廣告模板庫 → hard_block：直接禁言，不給自助驗證
     └─ 僅 @標籤 / 網址 / 非中文名稱 → 軟性可疑：發圖片驗證碼
```

**偵測類別（11 大類 269+ 範本，另有 L1 專屬規則涵蓋招募話術、成品帳號販售、代刷服務、假鈔泛化、包養招募等，不完全對應模板類別）：**

| 類別 | 說明 |
|------|------|
| A | 高收益招募（一天幾萬、提豪車） |
| B | 兼職詐騙（刷單、抖音養號、充電寶投放） |
| C | 洗錢 / 非法收款（USDT搬磚、微信代收） |
| D | 假鈔 / 違禁品（高仿鈔、VCC虛擬卡、GPT/Claude 金融卡詐騙） |
| E | TG推廣 / 廣告代發 |
| F | 博彩 / 幣圈（紅單、合約喊單、收米） |
| G | 色情 / 性服務 |
| H | 上門服務 / 便民引流（吉隆坡、東南亞） |
| I | NFT / 數位資產 |
| J | 遠程控制 / 監控工具 |
| K | 模糊引流 / 其他 |

## 環境要求

- Python 3.12+
- Linux/Mac 操作系統
- 建議安裝 `fonts-dejavu-core`（或 Liberation 字型），圖片驗證碼會優先使用系統字型，沒裝也不影響功能，純粹是好不好看的差異：
  ```bash
  apt install -y fonts-dejavu-core
  ```

## 快速安裝（Linux 環境）

### 1. 克隆倉庫

```bash
git clone https://github.com/1743986520/telegram-admin-bot.git
cd telegram-admin-bot
```

### 2. 執行安裝腳本

```bash
chmod +x install.sh
./install.sh
```
腳本會互動詢問 Bot Token（跟 @BotFather 申請）與你的 Telegram ID（跟 @userinfobot 查詢），並自動建立 Python 虛擬環境、安裝依賴、設定 systemd 服務。

### 3. 啟動

安裝腳本會處理好 systemd 服務，也可以手動啟動測試：

```bash
export BOT_TOKEN="你的Token"
export OWNER_ID="你的TelegramID"
venv/bin/python main.py
```

### 4. 更新依賴

`git pull`（或群組內 `/update`）只會拉程式碼，**不會**自動安裝新增的 pip 套件。`requirements.txt` 有變動時（例如新增 `Pillow`）記得手動跑一次：

```bash
venv/bin/pip install -r requirements.txt
systemctl restart telegram-admin-bot   # 或你自訂的服務名稱
```

## 圖片驗證碼流程

入群時若被判定為「軟性可疑」（`@標籤`、網址、非中文名稱），會收到一張帶干擾線的算術題圖片與 4 個選項按鈕：

- 答對：解除禁言、發送歡迎訊息，題目訊息 5 秒後自動刪除
- 答錯但未達上限：舊題目直接刪除，換一題新的，最多 3 次機會
- 答錯達上限：提示需管理員手動處理，8 秒後自動刪除
- 超過 30 分鐘未作答：視為過期，5 秒後自動刪除

若用戶名／暱稱／簡介**直接命中廣告模板庫**（比對邏輯與訊息廣告偵測相同），視為高置信度廣告帳號，會直接禁言且**不提供任何驗證方式**，只能由管理員用 `/ban` 或群組工具手動處理。

## 炸群應急鎖群（/omg）與防護模式（/guard）的差異

兩者都是「先禁言再說」的應急工具，但用途不同：

| | `/omg` 鎖群 | `/guard` 防護模式 |
|---|---|---|
| 適用情境 | 正在被炸群，需要立刻止血 | 想觀察一段時間再決定去留 |
| 入群提示 | 完全安靜，並自動刪除「OO加入了群組」系統訊息 | 完全安靜，不發送任何訊息 |
| 名單記錄 | 不記錄 | 記錄期間加入的所有人，存進 `known_groups.json`，重啟不丟失 |
| 解除時 | 純文字提示，恢復正常入群檢測流程 | 彈出「全部踢出／挑選踢出／不踢出」按鈕，挑選踢出可逐一勾選 |
| 共用解除指令 | `/stop`（僅本群管理員） | `/stop`（僅本群管理員） |

`/stop` 若兩者同時啟動會一併處理。挑選踢出的名單如果人數很多（幾十人以上），按鈕清單會很長，目前沒有做分頁，人數太多建議直接用「全部踢出」。

## 新增廣告範本

在 `ad_templates.py` 的對應類別加入新範本：

```python
# G. 色情 / 性服務類
"新範本內容",
```

若是規則性很強的話術（例如固定句型、關鍵詞組合），建議改在 `ad_detector.py` 的 `_RULES` 加正則規則，準確度更高也不吃 TF-IDF 相似度門檻。兩者修改後重啟 Bot 即時生效，無需其他操作；`/cleanupads`、`/addsample`、`/whitelist` 等指令內部會呼叫 `_reload_detector()`，不需重啟即可讓動態樣本生效。

## 規避手法防護

`ad_detector.clean_text()` 會在比對前先做這些正規化，抵銷常見的規避手法：

- 移除零寬字元（ZWSP / ZWNJ 等 Unicode Cf 類字元）
- NFKC 全形轉半形
- 還原字元間的拆字符號混淆，例如 `广+告+代+发`、`看/主/叶`、`最.牛.最.稳`
- 同音／形近字混淆表（`_CONFUSE_MAP`），例如 `Vx`→`微信`、`砍主頁`→`看主頁`、`憸`→`險`

新增混淆詞或拆字符號時，直接編輯 `ad_detector.py` 開頭的 `_CONFUSE_MAP` 或 `clean_text()` 裡的正則字元類即可。

## 群組功能開關

設定會持久化到 `known_groups.json`，舊群組未記錄的項目預設為啟用。可用名稱如下：

- `welcome`：入群歡迎
- `leave_notice`：離群通知
- `profile_check`：入群簡介／用戶名／暱稱檢測（含模板庫比對、非中文名稱判斷）
- `ad_detection`：訊息廣告檢測
- `ad_delete`：廣告自動刪除
- `ad_mute`：廣告自動禁言
- `ad_notify_admins`：廣告通知管理員
- `referendum`：`/vote` 全員禁言公投
- `proposals`：`/propose` 自訂提案
- `banme`：`/banme` 彩蛋禁言
- `ban_command`：`/ban` 管理員禁言

例如：`/feature ad_notify_admins off`。廣告偵測、刪除、禁言與通知是獨立開關，可按群組需求分開調整。`lockdown`（/omg）與 `guard_mode`（/guard）狀態也存在同一份 `known_groups.json`，但不透過 `/feature` 開關調整，只能用對應指令控制。

## 文件結構

```
telegram-admin-bot/
├── main.py           # 主程式、指令與事件處理
├── ad_detector.py    # 廣告偵測核心（clean_text 前處理 + L1 + L2 + 帳號畫像）
├── ad_templates.py   # 基礎廣告模板庫（11 大類，269+ 條）
├── ad_samples.py     # 動態廣告／白樣本庫讀寫，含與原生模板的正規化去重
├── custom_ad_samples.json  # 執行時產生，勿提交
├── whitelist_samples.json   # 執行時產生，勿提交
├── known_groups.json  # 執行時產生，各群組功能開關與鎖群/防護模式狀態
├── requirements.txt  # 依賴套件（含 Pillow，圖片驗證碼用）
├── install.sh        # 安裝腳本
└── runtime.txt       # Python 版本聲明
```

## 已知限制

- `/guard` 挑選踢出的名單沒有分頁，人數非常多時操作會不方便
- `known_groups.json`、`custom_ad_samples.json`、`whitelist_samples.json` 是執行期產生的資料檔，理論上不該進 git 追蹤；如果你的倉庫歷史裡它們曾經被 commit 過，`.gitignore` 不會讓 git 自動停止追蹤既有檔案，需要手動 `git rm --cached <檔案>` 一次
- 圖片驗證碼字型依賴系統是否裝了 DejaVu / Liberation，沒裝會退回 Pillow 內建字型（能用但較陽春）
